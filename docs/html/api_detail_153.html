1.接口说明

        获得上传文件的大小与状态，用以进行断点续传。

2.接口URL

        http://api.cloud.189.cn/getUploadFileStatus.action

3.支持格式

        xml

4.HTTP请求方式

        GET

5.请求Headers


字段标识 说明 数据类型 长度(字节) 可空
AccessToken OAuth2 Access Token String N
Signature 请求签名 String N
Date 请求日期和时间。例如：Wed, 01 Mar 2009 12:00:00 GMT String N

6.请求参数


字段标识 说明 数据类型 长度(字节) 可空
uploadFileId 上传文件Id Long N

7.请求样例 

GET /getUploadFileStatus.action?uploadFileId=29909576 HTTP/1.1

Host: api.cloud.189.cn

Date: date

AccessToken: key

Signature: signature

8.返回参数


字段标识 说明 数据类型 长度(字节) 可空
uploadFile 上传文件元素容器 Container N
—uploadFileId 用于断点续传的临时文件Id Long
—size 已经上传的文件大小，上传文件按照前一次上传时客户端的分块顺序保存，续传时要保证顺序与前一次一致,即续传的前提是客户端在上传与续传的期间不能对文件作出修改，否则应该以新上传格式重新上传。 Long
—fileUploadUrl 文件上传URL String
—fileCommitUrl 确认文件上传完成URL String
—fileDataExists 0或空 – 文件数据不存在 1 – 文件数据已存在，无需上传数据，可以确认上传完成 Integer

9.返回样例 

正确返回样例：			299095760http://upload.cloud.189.cn/uploadFileData.actionhttp://upload.cloud.189.cn/commitUploadFile.action


错误返回样例：无
